# 08 · 验证证据与边界

## 8.1 无硬件自动验证

仓库提供两层不依赖 ROS2、GPIO、雷达和 IMU 的验证：

```bash
python3 -m unittest discover -s tests -v
python3 tools/simulate_subsystems.py
```

单元测试直接覆盖实际节点复用的纯逻辑模块：差速轮速分解、精确圆弧里程计、速度源
优先级、锁存急停和路线校验。

子系统仿真使用一阶电机模型，在第 4 秒注入 30% 电压下降和额外负载，对比开环与 PID
闭环的稳态速度误差；同时验证急停 60 秒后仍锁存、复位会清除旧指令、里程计直线/圆弧
积分以及路线与地图指纹绑定。仿真输出 JSON，任一阈值失败都会以非零状态退出。

## 8.2 仿真能证明什么

- 控制、安全和里程计核心数学按照预期运行；
- 急停不会因速度源超时而自行解除；
- 未标定或绑定错误地图的路线不能启动巡逻；
- CI 可以持续发现这些逻辑的回归。

## 8.3 仿真不能证明什么

- GPIO 引脚、编码器相位、IMU 串口协议和雷达安装是否正确；
- 实车轮胎打滑、机械间隙、L298N 压降和树莓派调度抖动；
- PID 参数在真实负载下的超调、调节时间与稳态误差；
- AMCL 是否在具体场地长期不丢定位；
- 多圈巡逻成功率和真实避障距离。

上述项目必须按 `docs/06_build_run.md` 在实车上采集。`patrol_node` 会将每圈结果写入
`~/.ros/raspi_car/logs/patrol_metrics.jsonl`，简历中的成功率、单圈耗时、定位误差等数字
必须从这些实测记录计算，不能使用子系统仿真数字替代。

实车测试完成后汇总：

```bash
python3 tools/summarize_patrol_metrics.py
```

## 8.4 建议实车验收门槛

这些是测试门槛，不是当前已取得的成绩：

- 连续 20 圈巡逻，成功率不低于 95%；
- 任意时刻触发 E-STOP，电机保持停止直到人工复位；
- 低电量与额定负载下，左右轮稳态速度误差均不高于 0.03 m/s；
- 1 m 直线往返累计里程误差不高于 5%；
- 静态障碍物进入停止区后不发生接触。
